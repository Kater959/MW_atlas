# ATLAS_METALWORK_PROJECT_MEMORY.md

## Назначение файла

Этот файл — проектная память и ограничитель курса для Cursor.

Главная задача: не дать проекту Atlas Metalwork снова скатиться в обычную презентацию, slide deck, SaaS landing, корпоративный сайт или набор красивых карточек.

Если в дальнейших задачах появится конфликт между новым запросом и этим файлом, сначала сверяться с этим документом. Если запрос противоречит концепции Atlas, явно предупреждать: **"Это уводит проект от основной идеи Atlas Metalwork"**.

---

# 1. Что такое Atlas Metalwork

Atlas Metalwork — это не презентация.

Atlas Metalwork — это интерактивная инженерная энциклопедия / бесконечный mental map canvas по направлениям Metalwork.

Главная форма продукта:

```text
интерактивный инженерный холст
→ направления Metalwork
→ типы решений
→ боли клиента
→ где применяется
→ как работает
→ какие вопросы задать
→ какие данные нужны для расчёта
→ медиа: фото / эскизы / чертежи / CAD-like viewer
→ CTA: показать проблемную операцию
```

Atlas должен ощущаться как закрытая инженерная система знаний, которую можно:
- открыть на встрече с заказчиком;
- показать на проекторе;
- использовать менеджеру как штаб продаж;
- использовать как визуальную энциклопедию направлений;
- отправить клиенту ссылкой;
- разворачивать от общей карты к конкретному инженерному решению.

# 2. Главная ошибка, которую нельзя повторять

Нельзя превращать Atlas Metalwork в основной slide deck.

Ранее был сделан route:

```text

```

с 6 fullscreen-слайдами. Технически это может остаться как **Present Mode**, но это НЕ главный продукт.


Запомнить:

```text

Deck = не основа продукта
```

Если пользователь просит "дизайн Atlas", "главный экран", "основной интерфейс", "приложение", "приблуда", "энциклопедия", "карта направлений" — делать не слайды, а интерактивный холст.

# 3. Визуальное направление

Утверждённый стиль:

```text
Boardroom Engineering Deck + CAD Technical Viewer
```

Это стиль подачи, а не требование делать презентацию.

Одна фраза:

```text
закрытая инженерная система Metalwork:
тёмный командный центр,
интерактивный mental map,
CAD-like модель,
грани,
линии,
выноски узлов,
дорогая инженерная подача.
```

Визуальное ощущение:
- инженерный командный центр;
- тёмный зал;
- КБ / CAD / technical viewer;
- крупная типографика;
- дорогой промышленный софт;
- не игрушка;
- не лендинг;
- не PowerPoint;
- не cyberpunk HUD.

Референсы по логике:
- Snapdeck — только как идея fullscreen/present режима: одна мысль на экран.
- Lambda360view — как ощущение CAD-viewer: грани, линии, вращение, техническая модель.

Важно:
Snapdeck не означает, что весь Atlas должен быть слайдами.
Lambda360view не означает, что нужно сразу делать тяжёлый WebGL.

# 5. Главный экран Atlas

Главный экран должен быть не deck, а большой интерактивный холст.

Пример логики:

```text
ATLAS METALWORK

ОТК НЕ ПРЕДОТВРАЩАЕТ БРАК.
ОН ФИКСИРУЕТ УБЫТКИ.

Брак рождается раньше —
в операции, которая оставила ошибке место.

[интерактивный canvas]

                        [Сварочная оснастка]
                                ╲
[Кантователи] —— [ METALWORK ] —— [Станочная зажимная оснастка]
                                ╱              ╲
                [Реверс-инжиниринг]          [Агрегатные станки]
                         │
              [Нестандартное оборудование]
```

Центральный узел:

```text
METALWORK
Инженерия там, где ошибка не должна иметь места.
```

Узлы первого уровня:
1. Станочная зажимная оснастка
2. Сварочная оснастка
3. Сборочная оснастка
4. Кантователи
5. Подъёмные и монтажные приспособления
6. Агрегатные станки
7. Нестандартное оборудование
8. Реверс-инжиниринг
9. Контрольные приспособления
10. Механизация ручных операций

# 6. Поведение /atlas

Route:

```text
/atlas
```

Поведение:
1. Пользователь видит большой интерактивный холст.
2. В центре — Metalwork.
3. Вокруг — направления.
4. Можно pan / zoom.
5. При клике на направление открывается detail panel.
6. Detail panel должна быть инженерной карточкой знания, а не рекламной карточкой.

Detail panel содержит:
- сильный заголовок;
- короткий hook;
- что это;
- зачем нужно;
- какую боль закрывает;
- типы решений;
- где применять;
- когда не применять;
- какие вопросы задать клиенту;
- какие данные нужны для расчёта;
- медиа placeholders;
- ссылка "Открыть направление";
- ссылка "Present Mode", если есть;
- CTA: "Показать проблемную операцию".

# 7. Content model

Создать или поддерживать:

```text
src/content/atlas.ts
src/content/directions.ts
```

Модель узла Atlas:

```ts
export type AtlasNodeType =
  | 'root'
  | 'direction'
  | 'solution'
  | 'pain'
  | 'media'
  | 'question';

export interface AtlasNodeContent {
  id: string;
  title: string;
  shortTitle?: string;
  type: AtlasNodeType;
  summary: string;
  strongPhrase?: string;
  parentId?: string | null;
  children?: string[];

  painPoints?: string[];
  solutionTypes?: {
    title: string;
    description: string;
    whenToUse?: string;
    whenNotToUse?: string;
  }[];

  whereUsed?: string[];
  questions?: string[];
  requiredInput?: string[];
  mediaPlan?: {
    type: 'photo' | 'sketch' | 'drawing' | 'render' | 'video' | 'cad';
    title: string;
    description?: string;
    placeholder?: boolean;
  }[];

  cta?: {
    label: string;
    description?: string;
  };

  directionRoute?: string;
  presentRoute?: string;
}
```




```text
Показать проблемную операцию
```

- Покажите операцию, которую пора перестать терпеть.

- Покажите деталь, которую каждый раз приходится ловить.
- Покажите участок, где результат зависит от человека сильнее, чем от технологии.
- Покажите операцию, где ошибка всё ещё возможна.

# 10. Первый direction: Станочная зажимная оснастка

ID:

```text
clamping-fixtures
```

UI label:

```text
DIR-01 · CLAMPING FIXTURES
Станочная зажимная оснастка
```

Главная мысль:

```text
Станок за миллионы бессилен,
если база держится на ощущениях.
```

Что это:
Станочная зажимная оснастка — это инженерная система базирования, упора и зажима детали, которая превращает ручную установку в повторяемый процесс.

Боль:
- ручная выверка;
- подкладки;
- перезажим;
- риск перекоса;
- зависимость от смены;
- нестабильное время установки;
- прижим "по ощущениям";
- брак, который рождается в установке.

Типы:
- механическая;
- пневматическая;
- гидравлическая;
- модульная;
- специальная.

CAD labels:
- BASE — задаёт положение детали.
- STOP — не даёт детали уйти.
- CLAMP — удерживает с понятным усилием.
- SUPPORT — убирает прогиб и вибрацию.
- FIXTURE PLATE — собирает операцию в одну систему.

# 11. Режимы продукта

Atlas должен иметь минимум три режима:

```text
/atlas
Explore Mode — главный интерактивный холст.

/directions/[id]
Knowledge Mode — подробная энциклопедическая страница направления.

/present/[id]
Present Mode — fullscreen-показ выбранного направления.
```

Present Mode не удалять, но не считать основой.

# 12. Проверка любого решения

Перед реализацией любого нового экрана задать вопросы:

1. Это усиливает Atlas как интерактивную инженерную энциклопедию?
2. Это помогает раскрыть направления Metalwork глубже?
3. Это помогает менеджеру объяснить клиенту боль и решение?
4. Это помогает клиенту понять, где у него производственный хаос?
5. Это выглядит как дорогой инженерный интерфейс, а не как сайт или PowerPoint?
6. Это сохраняет ощущение mental map / CAD / command center?
7. Это не превращает проект обратно в slide deck?

Если ответ на пункт 7 — "нет, превращает", остановиться и переделать.
